[데이터와 데이터분석가]

01 데이터(data)란?
- 일반적인 정의: 컴퓨터에서 처리하는 문자, 숫자, 소리, 그림 형태로 된 자료

- 의미가 생겨나기 시작-> 감정, 흥미, 반응, 참여-> 정보(info)
  ex) '온/오프라인 방문기록'- 단순사실(data)/ '1달에 3번 이상 방문했네'- 정보(info)

- 데이터는 기본적으로 통계학 이론을 근간으로 사용되는 경우가 절대다수
  기술통계: 모수에 관계없이 데이터가 가진 통계적 의미를 찾는 것.
    ex) 산술평균(mean), 표준편차(deviation), 모집단/모수(population), 표본(sample), 신뢰구간(CI)
  추론통계: 데이터 표본을 가지고 모집단의 특성을 추정하는 것.
    ex) 가설과 거설검증을 위한 실험
  데이터 마이닝: 데이터가 가진 패턴/특징 찾아내는 것.
    ex) 회귀분석과 같은 알고리즘-> ML/DL-> AI

- 데이터 분석 프로세스
  1. 데이터 생성
  2. 데이터 수집
  3. 데이터 가공 후 저장(ETL)
    ETL: 추출(Extract)/변환(Transform)/로드(Load)
  4. 데이터 분석 및 시각화(BI)



02 데이터를 다루는 사람들
- Data Engineer(=개발자)
  대용량의 데이터를 수집, 관리하고 필요한 데이터를 추출하는 역량
  도구: MySQL, Spark, MapReduce, Airflow, 프로그래밍..

- Data Scientist/ ML Engineer
  대용량 데이터를 추출, 처리가 가능하고 그로부터 인사이트 뽑는 역량
  도구: Apache Spark, Machine Learning, 프로그래밍, 통계학

- Data Annotator

- Data Analyst(=비즈니스 문제 해결, 기획)
  데이터를 통해 '문제'를 해결하는데 집중하는데 필요한 역량
  도구: SQL(데이터 조회/활용), 각종 통계도구, 도메인 지식



03 프로덕트 직군과 데이터 직군
- Product
  목표: 아무것도 없이 고객으로부터 시작해서 제품/서비스를 통해 고객으로 끝냄
  역활: 내가 중요하다고 생각하는 걸 설득하고 결과로 만들어 내는 역할

- Data
  목표: 수없이 펼쳐진 데이터의 바다에서 인사이트를 찾아내는 것 혹은 찾아내기 위해 필요한 업무
  역할: 데이터에서 의미를 발견해 개선/시도 방안에 대해 설득하는 역할 혹은 그럴 수 있게 체계를 구축하는 역할



04 고객데이터 분석, 왜 해야할까?
- Peter Drucker: 측정하지 않으면 관리할 수 없으며, 관리할 수 없으면 개선시킬 수도 없다.

- 서비스 데이터: 서비스 운영을 위한 모든 데이터
    가입, 결제 등 서비스 운영을 위해 반드시 필요한 데이터
  ex) 결제 내역, 회원가입 정보, 서비스 신청 내역 등
  서비스 주요 기능에 큰 영향을 미친다.

- 사용자 행동 데이터: 사용자들의 행동 패턴을 확인하기 위해 별도로 저장하는 데이터
  ex) 웹로그 데이터, 앱로그 데이터, 이벤트 데이터, 사용자 로그 데이터 등
  상대적으로 규모가 크고, 신뢰도는 낮다.
    누락, 중복 집계된 가능성이 높음
    페이지뷰, 스크롤, 클릭 등



05 사용자 행동패턴 분석방법
- 고객 대면 관찰

- 고객 인터뷰

- 사용자 행동 데이터 분석=> 런칭 이후가 시작점.
  사용자 행동 데이터 분석 주제
  ex) 사용자들이 기능을 어떤 순서로 사용하는가
      UI를 변경한 이후 사용자들의 행동 변화
      어느 기능과 특정 페이지에 관심이 많고 적은가
      결제한 사용자가 어떤 페이지에서 결제전환이 되었는가
      어떤 페이지에서 이탈을 하는가
  사용자 행동 데이터 분석시 주의사항
  1. 데이터가 기록되는 방식을 정확하게 이해해야 한다.
  2. 데이터는 언제나 잘못 기록될 가능성이 있다.
  3. 데이터를 조회할 때 주의가 필요하다.
