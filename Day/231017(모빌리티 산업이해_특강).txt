4. STAR 형식에 맞게 정리
  Situation: 내가 겪은 상황, 겉으로 드러난 그 당시 모습들
  Target: '문제'라고 인지 및 정의했던 것
  Action: 그 문제를 위해서 내가 했던 행동(들)
  Result: 행동으로 인해 어떤 결과가 나타났는지, 어떤 걸 배웠는지
          (앞으로/현재까지 어떻게 적용 할지/해왔는지)
5. 자소서 및 포트폴리오에 반영

01 데이터 분석 프로세스
  1. 데이터 생성
  2. 데이터 수집
  3. 데이터 가공 후 저장(ETL)

  4. 데이터 분석 및 시각화(BI)

02 데이터를 다루는 사람들
- Data Engineer(=개발자)
  대용량의 데이터를 수집, 관리하고 필요한 데이터를 추출하는 역량
  도구: MySQL, Spark, MapReduce, Airflow, 프로그래밍..

- Data Scientist/ ML Engineer
  대용량 데이터를 추출, 처리가 가능하고 그로부터 인사이트 뽑는 역량
  도구: Apache Spark, Machine Learning, 프로그래밍, 통계학

- Data Annotator

- Data Analyst(=비즈니스 문제 해결, 기획)
  데이터를 통해 '문제'를 해결하는데 집중하는데 필요한 역량
  도구: SQL(데이터 조회/활용), 각종 통계도구, 도메인 지식
